## Handoff Checklist

### PM

| 項目                                                                  | 說明                                                                                                                      |
| ------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| UX流程操作是否符合需求                                                        |                                                                                                                         |
| 是否符合當初要求的裝置版本                                                       |                                                                                                                         |
| 再次確認文字翻譯是否有遺漏或英翻爆版                                                  |                                                                                                                         |
| 需要下載的檔案是否需要翻譯？                                                      |                                                                                                                         |
| 提示文字、錯誤訊息是否都有示意圖製作                                                  |                                                                                                                         |
| 介面開關連動的限制（先開啟Ａ，才能開啟B）                                               |                                                                                                                         |
| 其他已存在頁面是否有共通類似的設計，是否遵循通用設計                                          |                                                                                                                         |
| <p>Mobile 和 App 流程、畫面差異 <br>文案用詞避免過於針對特定族群，或僅有少量文字變換 </p>           |                                                                                                                         |
| 除了正常的 ideal 狀態外，確認有支援 empty, partial, loading, complete, error 等狀態  |                                                                                                                         |
| 檢查不同裝置與解析度的畫面呈現                                                     |                                                                                                                         |
| 完整的介面說明文字                                                           |                                                                                                                         |
| 確認 UX handoff 的文字                                                   | <p></p><ul><li>文案用詞避免過於針對特定族群，或僅有少量文字變換，這樣可直接模組化如有文案還未定的地方，請先提示或給出對應長度，讓設計好處理</li><li>若功能有下載檔案，記得範例檔的文案要一併送翻譯</li></ul> |
| 票/figma上的機制是精準描述                                                    | 請避免把系統判斷方式寫得太籠統，這會導致查票時不確定具體設計                                                                                          |
| 翻譯需求                                                                | <p>管理者後台不翻譯，其他都要翻譯。<br>學習數據目前只翻一半。</p>                                                                                  |
| 跟運營窗口確認是否需要內部簡易說明或展示                                                |                                                                                                                         |

### Design

| 項目                             | 說明                                                                 | 狀態 |
| ------------------------------ | ------------------------------------------------------------------ | -- |
| 畫面的 status 確認                  | 除了正常的 ideal 狀態外，確認有支援 empty, partial, loading, complete, error 等狀態 |    |
| Ux handoff 格式確認                | <p>大功能用色塊區分</p><p>frame 命名不要重複，方便大家尋找</p>                          |    |
| 確認 UX Handoff 可以轉化到行動裝置上的設計與操作 | 如果行動裝置也會使用到，請確認是否由行動裝置的使用體驗出發來做設計（避免由桌機尺寸思考流程設計，並直接套用至行動裝置）        |    |
| 版面後續的規劃                        | 版面空間有限，設計要是可以容易擴充的樣式，並且在一定的時間內，不去更動可擴充數量的上限。                       |    |



### 前端

| 項目                     | 說明                                  | 狀態 |
| ---------------------- | ----------------------------------- | -- |
| 確認 UX handoff 有處理到極端情境 | 從開發角度，檢視 UX Handoff 有無缺漏，特別是極端情境    |    |
| 與 App 確認相同功能的邏輯是否一致    | 兩邊做到同樣功能的人，需要共同確認 web/app 的操作邏輯是一致的 |    |

### 後端

| 項目                     | 說明                                                                               | 狀態 |
| ---------------------- | -------------------------------------------------------------------------------- | -- |
| 確認 UX handoff 有處理到極端情境 | 從開發角度，檢視 UX Handoff 有無缺漏，特別是極端情境                                                 |    |
| 新舊相容                   | 開發新功能或需求時，不管需求怎麼說，開發時都應該要做新舊相容。在初期就以相容為目標，會比做到一半（或做完後）要追加補上相容簡單不少，出現 Bug 的機率也比較低 |    |
